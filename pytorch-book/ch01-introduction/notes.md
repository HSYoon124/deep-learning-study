# ✏️ Chapter 1. Introduction

## 1. AI, Machine learning, Deep learning

- AI 구현 방법으로 machine learning, deep learning이 있음
- 인공지능 > 머신러닝 > 딥러닝
  
|구분|Machine Learning|Deep Learning|
|:---:|:---:|:---:|
|개념|입력 데이터(인간이 전처리한 데이터)에 알고리즘을 적용해 예측 수행|대량의 데이터를 신경망에 적용해 컴퓨터 스스로 분석|
|재사용|불가능|가능|
|훈련시간|단시간|장시간|

즉, 입력 데이터 특성 추출 과정을 인간이 하냐, 컴퓨터가 하냐의 큰 차이점 존재.

---

## 2. Machine Learning이란?

- 인공지능의 한 분야
- 컴퓨터 스스로 대용량 데이터 특성을 추출해 학습, 예측 수행 (= 컴퓨터가 학습 가능한 알고리즘, 기술 개발)

2.1. 학습 과정 <br>
학습 단계(learning): 머신 러닝 알고리즘에 적용하여 학습시키고, 이 학습 결과로 모형이 생성 <br>
예측 단계(prediction):학습 단계에서 생성된 모형에 새로운 데이터를 적용하여 결과를 예측
<img width="500" height="364" alt="Image" src="https://github.com/user-attachments/assets/e15dae60-229e-42e1-a53b-e25eb7a35992" />

2.2. 주요 구성 요소
- 데이터: 학습 모델 생성에 사용, 훈련과 테스트 데이터셋으로 분리
- 모델: 머신 러닝 학습 단계에서 얻은 최종 결과뭉 (=가설), 모델 선택 후 학습 및 평가를 통해 업데이트함

2.3. 머신 러닝 학습 알고리즘
- 지도 학습: 정답 알려주고 학습
- 비지도 학습: 정답 알려주지 않고 특징 비슷한 데이터를 clusturing(범주화) 하여 예측하는 학습, 유사도 기반
- **강화 학습**: 자신의 행동에 대한 보상을 받으며 학습, 보상이 커지는 행동 빈도 증가하도록 학습 (12장 강화 학습에서 설명)

(각 학습에서 자주 사용되는 알고리즘은 3장 머신 러닝 핵심 알고리즘에서)

## 3. Deep Learning이란?

- 인간의 신경망 원리를 모방한 심층 신경망 이론을 기반으로 고안된 머신 러닝 방법

<img width="400" height="340" alt="Image" src="https://github.com/user-attachments/assets/851be2d7-391f-4bf9-8f75-67fda48dba38" />

3.1. 학습 과정

<img width="300" height="401" alt="Image" src="https://github.com/user-attachments/assets/bd748441-e7a8-4fc7-b7ca-b7654d522ff6" />

  1. 데이터 준비: [파이토치](https://tutorials.pytorch.kr/)나 [케라스](https://keras.io/)에서 제공하는 데이터셋을 사용,
     [캐글(Kaggle)2](https://www.kaggle.com/datasets) 같은 곳에 공개된 데이터를 사용
  2. 모델(모형) 정의: 신경망 생성. 은닉층 개수가 많을수록 성능이 좋아지지만 과적합이 발생할 확률이 높음
  3. 모델(모형) 컴파일: 컴파일 단계에서 활성화 함수, 손실 함수, 옵티마이저를 선택, 데이터 형태에 따라 다양한 옵션 선택, 과적합을 피할 수 있는 활성화 함수 및 옵티마이저 선택이 중요 <br>
     e.g. 훈련 데이터셋 형태가 연속형이라면 평균 제곱 오차(Mean Squared Error, MSE), 이진 분류(binary classification)라면 크로스 엔트로피(cross entropy)
  4. 모델(모형) 훈련: 한 번에 처리할 데이터양 지정, 한 번에 처리해야 할 데이터양이 많아지면 학습 속도가 느려지고 메모리 부족 문제를 야기할 수 있기 때문에 적당한 데이터양을 선택하는 것이 중요. <br>
     따라서 전체 훈련 데이터셋에서 일정한 묶음으로 나누어 처리할 수 있는 배치와 훈련의 횟수인 에포크 선택이 중요.
     훈련 과정에서 값의 변화를 시각적으로 표현하여 눈으로 확인하면서 파라미터와 하이퍼파라미터에 대한 최적의 값을 찾을 수 있어야 함.
     <img width="400" height="294" alt="Image" src="https://github.com/user-attachments/assets/433acf55-eb28-4eb7-bfa3-139eba7cfe31" />

> 성능이 좋다는 의미는?  예측을 잘한다, 훈련 속도가 빠르다

  5. 모델(모형) 예측: 검증 데이터셋을 생성한 모델(모형)에 적용하여 실제로 예측을 진행. 이때 예측력이 낮다면 파라미터를 튜닝하거나 신경망 자체를 재설계해야 할 수도 있음.

3.3. 딥러닝 학습 알고리즘
- 지도 학습:
  1) 합성곱 신경망 - 컴퓨터 비전에서 가장 많이 사용 / 목적에 따라 이미지 분류, 이미지 인식, 이미지 분할로 분류
  2) 순환 신경망 - 시간에 따른 데이터가 있을 때 사용(e.g.주식 그래프) / <br>
     단점: 역전파 과정에서 기울기 소멸 문제가 발생 -> 해결책: 게이트(gate)를 세 개 추가한 LSTM(Long Short-Term Memory) / 망각 게이트(과거 정보를 잊기 위한 게이트), 입력 게이트(현재 정보를 기억하기 위한 게이트)와 출력 게이트(최종 결과를 위한 게이트)를 도입하여 기울기 소멸 문제를 해결, 시계열 문제에서 가장 활발히 사용

- 비지도 학습:
  1) 워드 임베딩 - 단어(자연어)를 벡터로 표현 / 워드투벡터(Word2Vec)와 글로브(GloVe)를 가장 많이 사용
  2) 군집 - 아무런 정보가 없는 상태에서 데이터를 분류하는 방법 / <br> 목표: 한 클러스터 안의 데이터는 매우 비슷하게 구성하고 다른 클러스터의 데이터와 구분되도록 나누는 것 / 머신 러닝과 함께 사용하면 좋음

- 전이 학습: <br>
  사전에 학습이 완료된 모델(pre-trained model)(사전 학습 모델)을 가지고 우리가 원하는 학습에 미세 조정 기법을 이용하여 학습 <br>
  사전에 학습이 완료된 모델이 필요하며, 학습이 완료된 모델을 어떻게 활용하는지에 대한 접근 방법이 필요
